# Linux AppImage IME Regression 1-Pager

## Background

HOP Linux AppImage v0.1.7 still fails Korean input for a CachyOS KDE Plasma Wayland user, while the deb/rpm install path works. The v0.1.7 AppImage `AppRun` generated by linuxdeploy forces `GDK_BACKEND=x11` and `GTK_IM_MODULE_FILE=$APPDIR/usr/lib/x86_64-linux-gnu/gtk-3.0/3.0.0/immodules.cache`.

## Problem

The bundled AppImage GTK cache only contains built-in modules such as `xim` and `wayland`; it does not contain host IME modules such as `fcitx`, `ibus`, or `kime`. The previous runtime fix treated any existing `GTK_IM_MODULE_FILE` as a user override and returned early, so the AppImage-provided cache was never replaced.

## Goal

Make the AppImage use the host GTK IM module cache when the active cache is the AppImage-bundled cache and the host cache provides the IME module requested by, or inferred from, the user's environment.

## Non-goals

Do not modify upstream `third_party/rhwp`. Do not remove the linuxdeploy `GDK_BACKEND=x11` workaround in this patch. Do not force a single IME globally when the user has provided an explicit custom cache.

## Constraints

Keep behavior scoped to Linux AppImage runtime. Preserve user-provided `GTK_IM_MODULE_FILE` values outside `APPDIR`. Support Debian/Ubuntu, Fedora/openSUSE, and Arch/CachyOS cache paths.

## Implementation outline

Detect whether `GTK_IM_MODULE_FILE` points inside `APPDIR`. Treat that as AppImage-owned, not user-owned. Infer the requested module from `GTK_IM_MODULE`, `XMODIFIERS`, `QT_IM_MODULE`, `SDL_IM_MODULE`, or `INPUT_METHOD`, normalize `fcitx5` to `fcitx`, and point GTK at a host cache containing the module. Also prepend the host GTK root to `GTK_PATH` so relative cache entries can resolve on Arch/CachyOS.

## Verification plan

Add Rust unit tests for AppImage-owned cache detection, explicit user override preservation, `fcitx5` normalization, `XMODIFIERS` parsing, host cache selection, and `GTK_PATH` merging. Run `cargo test` for `apps/desktop/src-tauri`.

## Rollback

Revert the runtime fix changes in `apps/desktop/src-tauri/src/linux_runtime.rs`; deb/rpm behavior is unaffected because the code only runs under AppImage.
